## Directory Purposes

**Root:**
- Purpose: Build system and project configuration
- Key file: `build.py` — orchestrates all asset transformation

**`assets/`:**
- Purpose: Source components for HTML generation
- Contains: HTML sections, CSS, modal data (JSON), fonts (base64), background images
- Process: Build.py reads all files, applies transformations, assembles into single HTML

**`assets/fonts/`:**
- Purpose: MG brand typography as base64-encoded files
- Files: 5 font families × weight variants
- Reason: Embed fonts in HTML to avoid external CDN requests

**`assets/bg_photos/`:**
- Purpose: Background images for sections
- Files: ~10 JPEGs
- Process: Referenced in CSS, copied to output/img/ during build

**`output/`:**
- Purpose: Generated website
- Generated by: `build.py`
- Files: Single monolithic `index.html` (1.4MB, ~7000 lines)
- Deployment: This directory is served (e.g., via HTTP)

**`docs/`:**
- Purpose: Versioned documentation and planning
- Contains: Prior analysis, planning documents
- Not regenerated by build

## Naming Conventions

**Files:**

- Python build script: lowercase with underscore (`build.py`)
- HTML/JSON components: lowercase with underscore (`sections.html`, `modals_data.json`)
- CSS: lowercase with underscore (`styles.css`)
- Fonts: PascalCase family + underscore + weight (`Favorit_w400.b64`)
- Images: descriptive lowercase (`hero.jpg`, `contexto.jpg`)

**Directories:**

- Source code: lowercase (`assets`)
- Output: lowercase (`output`)
- Built content: lowercase (`docs`)

**HTML IDs (Section Navigation):**

- Pattern: kebab-case, descriptive, single word or two
- Examples: `#hero`, `#contexto`, `#campo`, `#convocatoria`, `#incentivos`, `#locaciones`, `#catering`, `#tecnica`, `#moderadores`, `#entregables`, `#inversion1`, `#dolor`, `#intelligence`, `#comparativa`, `#cierre`

**Modal IDs:**

- Pattern: `m-` prefix + category + descriptor
- Examples:
  - Observations: `m-conv`, `m-micro`, `m-catering-obs`, `m-estructura`
  - Venues: `m-v1` through `m-v6`
  - Moderators: `m-mod1` through `m-mod5`
  - Steps: `m-step1` through `m-step4`
  - Intelligence: `m-lav`, `m-360`, `m-diar`, `m-dash`

**CSS Classes:**

- Pattern: kebab-case, descriptive
- Examples: `modal-overlay`, `modal-box`, `.anim`, `.anim-d1`, `.counter-grid`, `.section-title`, `.nav-link`
- Utilities: `.bg-hero`, `.bg-secondary`, `.bg-card`, `.text-primary`, `.text-muted`

**Data Attributes:**

- `data-modal="[id]"`: Opens modal with given ID
- `data-close="[id]"`: Closes modal with given ID (deprecated, now uses class selector)
- `data-count="[number]"`: Value for count-up animation
- `data-idx="[index]"`: Carousel slide index
- `data-mod="[index]"`: Moderator carousel slide index

## Where to Add New Code

**New Section (Part I or Part II):**
1. Add `<section id="[new-id]">...</section>` to `assets/sections.html`
2. Add corresponding CSS rules to `assets/styles.css`
3. Add nav link to nav-links div in sections.html: `<a href="#[new-id]" class="nav-link">`
4. Rebuild: `python3 build.py`

**New Modal:**
1. Add entry to `assets/modals_data.json` with key `m-[name]`
2. HTML structure: `{"m-name": "<div class=\"modal-overlay\" id=\"m-name\"><div class=\"modal-box\">...</div></div>"}`
3. Reference in sections: Add `data-modal="m-name"` to clickable element
4. Rebuild: `python3 build.py`

**Styling Changes:**
1. Edit `assets/styles.css` directly
2. Rebuild: `python3 build.py`
3. No CSS duplication — build.py deduplicates `.mod-mosaic` blocks

**JavaScript Behavior:**
1. Edit `build_js()` function in `build.py` (lines 157-517)
2. Insert code into the triple-quoted string
3. Rebuild: `python3 build.py`

**New Font:**
1. Add .b64 file to `assets/fonts/`
2. Add entry to `assets/font_declarations.json`
3. Rebuild: `python3 build.py`

**New Background Image:**
1. Add JPG to `assets/bg_photos/`
2. Reference in CSS: `background-image:url('/img/[filename].jpg')`
3. Rebuild: `python3 build.py` (copies images to output/img/)

## Special Directories

**`assets/`:**
- Purpose: All source material for HTML generation
- Generated: No
- Committed: Yes
- Modification: Edit directly; rebuild always runs full assembly

**`output/`:**
- Purpose: Generated website
- Generated: Yes (by `python3 build.py`)
- Committed: Yes (versioned HTML)
- Modification: Do not edit; regenerate via build

**`.planning/codebase/`:**
- Purpose: Architecture analysis documents
- Generated: No (manually maintained)
- Committed: Yes
- Modification: Update for architecture changes

## Build Process Workflow

1. **Clean state:** Read fresh copies of all assets
2. **Font embedding:** Generate @font-face rules from base64 font files
3. **Content fixes:** Apply pricing transformations, convert modals, deduplicate CSS
4. **Assembly:** Inject all transformed assets into single HTML template
5. **Output:** Write `output/index.html` (single file, ~1.4MB)
6. **Asset copying:** Copy background images to `output/img/`
7. **Stats:** Log file size, line count, onclick references, data-modal triggers

**Idempotent:** Running `python3 build.py` multiple times produces identical output.
